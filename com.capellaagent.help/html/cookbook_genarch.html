Recipe: Generative Architecture

Week 17-22 This recipe uses the Architect agent mode and five tools shipping in Weeks 17-22: propose_architecture, suggest_decomposition, find_missing_traces, generate_allocation_proposal, and review_architecture_diff.

Step 1 — Switch to Architect Mode

/mode architect

The system prompt switches to architect persona — the agent now reasons about components, allocations, and system decomposition instead of fault triage or exploration.

Step 2 — Propose an Architecture from Requirements

Paste a set of requirements as context:

Propose an architecture for these requirements:
1. The system shall provide real-time video streaming to all passenger seats.
2. The system shall support a minimum of 10,000 hours of on-demand content.
3. The system shall allow crew to override passenger content with safety announcements.
4. The system shall operate within 200W total power budget.
5. The system shall be maintainable without removing passenger seats.

The agent calls propose_architecture and returns a structured proposal:

  • A list of proposed logical components (e.g., Content Server, Display Unit, Crew Control Panel, Power Distribution Module).
  • Proposed function allocations for each component.
  • Key interfaces between components.
  • Rationale for each design decision, referencing the input requirements.

Step 3 — Review the Diff Preview

The proposal is presented as a diff — additions shown in green, nothing removed until you confirm. Review each proposed element and click Accept or Reject on any item.

When you accept the full proposal, the agent calls the appropriate write tools (create_element, allocate_function, create_interface) to commit the changes to the model.

Step 4 — Find Missing Traces in the Avionics Layer

Find missing traces in the Avionics layer

The agent calls find_missing_traces for the Avionics System subcomponents and returns:

  • Physical components with no functional allocation.
  • System functions not realized in the Logical Architecture.
  • Operational activities not linked to System functions.

Step 5 — Suggest Decomposition

Suggest a decomposition for the In-Flight Entertainment System

The agent calls suggest_decomposition with the IFE system as the target. It analyzes the current functional allocation and suggests a decomposition into sub-components that respects separation of concerns, maintainability, and the power budget constraint (if captured in the model).

Step 6 — Generate Allocation Proposal

Propose allocations for the unallocated functions in the PA layer

The agent calls generate_allocation_proposal, which queries all unallocated physical functions and suggests which component each should be allocated to based on naming, co-location, and exchange patterns.

Iterative workflow: Architect mode works best iteratively. Accept a subset of the proposal, validate in the model, then ask for the next iteration. Use Ctrl+Z liberally — every change is reversible.
