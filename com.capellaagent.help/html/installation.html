Installation Guide

Prerequisites

RequirementVersionDownload
Eclipse Capella7.0.1eclipse.org/capella
Java JDK17+adoptium.net (Temurin)
Apache Maven3.8+maven.apache.org (build only)
Git2.xgit-scm.com (build only)

Verify your Java version: open a terminal and run java -version. If Capella launches with a bundled JRE, ensure it is also Java 17 by checking Help → About Capella → Installation Details → Configuration.

Option A: Build from Source with reinstall.bat

This is the recommended path for developers and early adopters.

Step 1 — Clone the repository

git clone https://github.com/your-org/capella-agent.git
cd capella-agent

Step 2 — Generate the local Capella p2 repository

Capella's OSGi bundles are not on Maven Central, so the build references a local p2 repository generated from your Capella installation. 

# Windows (from the Capella installation directory)
# The p2 repo is usually already present at:
#   C:\Apps\capella-7.0.1\p2repo
# If it is missing, see tools/README.md for generation steps.

Step 3 — Configure CAPELLA_HOME in reinstall.bat

Open reinstall.bat and edit the CAPELLA_HOME variable:

set CAPELLA_HOME=C:\Apps\capella-7.0.1

Step 4 — Run the build

reinstall.bat

This runs mvn clean verify then copies all plugin JARs into Capella's dropins/ folder.

Step 5 — Restart Capella

Close and reopen Capella. The plugin loads automatically from the dropins folder.

Option B: Maven Build Only (no reinstall.bat)

mvn clean verify -pl com.capellaagent.site -am

The built p2 update site is generated at com.capellaagent.site/target/repository/. Use this path as the update site URL in the next option.

Option C: Install New Software (GUI)

  1. Build the p2 site using Option B, or obtain a pre-built ZIP from GitHub Releases.
  2. In Capella: Help → Install New Software...
  3. Click Add...
  4. Enter a name (e.g., Capella Agent) and click Local... to browse to com.capellaagent.site/target/repository or Archive... for a ZIP file.
  5. Select Capella Agent - AI Multi-Agent Ecosystem.
  6. Click Next, review the items, accept the EPL 2.0 license, and click Finish.
  7. Restart Capella when prompted.
SmartScreen warning (Windows): Because the JARs are not yet code-signed, Windows SmartScreen may block the installation. Right-click the downloaded ZIP, select Properties, and click Unblock. See Known Limitations #8.

Verifying the Installation

  1. In Capella, open Help → About Capella → Installation Details.
  2. On the Plug-ins tab, search for com.capellaagent.
  3. You should see all 9 bundles listed with version 1.0.0.qualifier.
  4. Open the AI Model Chat view with Ctrl+Shift+M. The view should appear with the welcome wizard or an empty chat state.

Common Installation Errors

ErrorCauseFix
Cannot resolve dependency org.polarsys.capella.core.data.gen The Capella p2 repository is not configured or the path is wrong Check CAPELLA_HOME in reinstall.bat and verify the p2repo folder exists
mvn: command not found Maven is not on PATH Add Maven's bin/ directory to your system PATH, or use the full path
Capella starts but no AI Model Chat view in the menu The plugin JARs were not copied to dropins/ Run reinstall.bat again and check for errors in the Maven output
Java version error during build Maven is using Java 11 or earlier Set JAVA_HOME to a Java 17+ JDK before running the build
