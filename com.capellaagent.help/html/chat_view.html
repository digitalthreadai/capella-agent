The AI Model Chat View

The AI Model Chat view is the primary interface for interacting with Capella Agent. It is an Eclipse view that docks anywhere in the Capella workbench.

Toolbar Elements

ElementDescription
Mode selector Dropdown to switch between agent modes. Changing mode resets the system prompt but preserves conversation history (you can still follow up on prior results).
Project selector Dropdown listing all open Capella sessions. Select the project you want the agent to work with. Only one project is active at a time.
Provider / Model selector Dropdown showing the currently active provider and model (e.g., "Claude / claude-opus-4-5"). Click to switch mid-conversation.
Clear Clears the conversation history and resets the browser area to the empty state. Equivalent to the /clear slash command.
Export Exports the current conversation as an HTML file or Markdown file to your chosen directory.
Detach Opens the chat in a standalone window, useful for multi-monitor setups where you want the chat on one screen and the Capella model on another.
Theme toggle Switches the chat area between light and dark themes. The default is light.
Help Opens this help documentation.

Browser Area

The browser area uses an embedded web renderer (SWT Browser / WebKit). All agent responses are rendered as HTML, which means:

  • Tables are rendered with full HTML formatting and are sortable.
  • Code blocks use syntax highlighting.
  • Tool call indicators appear as styled inline blocks.
  • UUIDs in results are clickable hyperlinks that trigger Eclipse navigation.
  • Right-click anywhere in the browser to access standard browser context menu options (copy, select all, view source).

Input Area

The input area at the bottom contains:

  • Text field: Multi-line input. Press Enter to send, Shift+Enter for a new line.
  • Send button: Submits the query. Disabled while a response is streaming.
  • Token counter: Live estimate of the current context size vs. the provider's context window limit. The estimate uses a character-density heuristic; see Known Limitations #2.
  • Cost display: Running session cost estimate based on the provider's published token prices. Shows session aggregate (not per-turn). See Known Limitations #3.

Conversation Flow

  1. You type a query and press Enter.
  2. The agent sends the conversation history plus your query to the LLM.
  3. If the LLM decides a tool call is needed, the tool call indicator appears and the tool runs against the live Capella model.
  4. Tool results are fed back into the conversation automatically (the agent may call multiple tools in sequence for complex queries).
  5. The LLM composes a final response, which streams into the browser area.
  6. You can ask follow-up questions; the agent remembers the full conversation context.
Cancellation: Press Esc while a query is running to cancel the in-flight request. Partially completed tool calls may have already made model changes — use Edit → Undo (Ctrl+Z) to reverse them.

Write-Operation Consent

When the agent is about to modify your Capella model — create, rename, delete, or apply an architecture proposal — it pauses and shows a native consent dialog before any write occurs. You cannot accidentally change your model by typing a query.

Consent dialog layout: The dialog shows the tool name and category badge (MODEL_WRITE, DESTRUCTIVE, or REQUIREMENTS), the full argument set in read-only monospace text, and an "LLM reasoning (untrusted)" section in amber that contains a short excerpt of why the agent wants to make the change.

The Deny button is focused by default — pressing Enter without moving the cursor denies the action. Closing the dialog with X also denies.

A "Remember my choice for this tool in this session" checkbox appears for ordinary model writes, letting you approve a tool type once per session. The checkbox is disabled for destructive operations (delete, merge, apply architecture diff) — those always prompt, every time.
The "LLM reasoning" text is untrusted. It is capped at 300 characters and rendered as plain text. An adversarial prompt inside a requirement or diagram label cannot embed HTML or control characters to make the reasoning look official. Treat it as a hint, not as a guarantee.

If a write is denied, the agent receives a denial message and may rephrase its request or stop. The denial is recorded in the audit log at {workspace}/.capella-agent/audit.log. See Security & Governance for more on the audit trail.
