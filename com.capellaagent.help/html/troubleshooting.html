Troubleshooting

This page covers common issues and their solutions. If you encounter a problem not listed here, please file a GitHub issue with the log output from Window → Show View → Error Log.

Common Issues

Error: "Multiple Capella projects are open"

Symptom: The agent replies that it cannot determine which project to use, or the result table shows elements from the wrong project.

Cause: You have more than one Capella project open with an active Sirius session.

Fix: Use the project dropdown in the chat view toolbar to explicitly select the project you want the agent to work with.

Error: "No open Capella sessions found"

Symptom: The agent replies that it cannot find any open Capella models.

Cause: No Capella project has an active Sirius session. Either no project is open, or the .aird file has not been opened.

Fix: Double-click the .aird file in the Project Explorer to open the Sirius session. The status indicator in the chat view toolbar should change from red to green.

Error: "Could not authenticate" / 401 Unauthorized

Symptom: The agent sends a query but returns an authentication error.

Cause: The API key for the active provider is missing, expired, or incorrect.

Fix:

  1. Open Window → Preferences → Capella Agent → LLM Providers.
  2. Select the active provider.
  3. Re-enter the API key (keys must be re-entered after Capella restarts due to OS keystore access policies on some machines).
  4. Click Test Connection. A green checkmark confirms the key works.
  5. Click Apply and Close.

Error: "Context length exceeded"

Symptom: The agent returns an error stating the conversation is too long, or the provider API returns a 400 error referencing context length.

Cause: The accumulated conversation history plus the model results exceeds the provider's context window limit.

Fix options:

  • Type /clear to reset the conversation and start a fresh session.
  • Switch to a provider with a larger context window. Claude (Anthropic) supports up to 200k tokens; OpenAI GPT-4o supports 128k tokens; Groq supports up to 32k tokens.
  • Use more targeted queries instead of listing all elements — ask for specific layers, types, or use the max_results limit.

Error: "Tool not available" / Hallucinated tool name

Symptom: The agent attempts to call a tool that does not exist (e.g., get_all_model_data or search_model) and reports that the tool is not available.

Cause: The LLM hallucinated a tool name. This happens more frequently with smaller models or when the query uses vocabulary that does not match any real tool.

Fix: Rephrase your query using more specific vocabulary. For example:

  • Instead of "search the model for avionics" → "List all components in the PA layer whose name contains Avionics"
  • Type /tools to see the full list of available tool names for reference.
  • Use a more capable model (e.g., Claude Opus or GPT-4o) for complex multi-step queries where smaller models tend to hallucinate.

Performance: Slow First Query

Symptom: The first query after opening Capella Agent takes 5–15 seconds before the LLM starts responding.

Cause: The first call initializes the HTTP client, warms the JVM JIT compiler, and (for some providers) establishes a TLS session.

Fix: Send a simple warm-up query when you first open the chat view, such as "Hello" or "/version". Subsequent queries will be significantly faster.
