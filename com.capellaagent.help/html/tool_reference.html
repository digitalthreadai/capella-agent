Tool Reference

Capella Agent ships with 95+ tools across 9 categories. Tools are invoked automatically by the LLM based on your natural language queries — you rarely need to name them explicitly. This reference is useful when you want to understand what is available or when debugging unexpected behavior.

For the full tool details with all parameters, see TOOLS.md in the repository root.

Model Read Tools (21)

ToolDescriptionKey Parameters
list_elements List model elements by ARCADIA layer and type. Returns name, UUID, type, description. layer (oa/sa/la/pa), element_type (functions/components/actors/exchanges/capabilities/all), max_results
get_element_details Get full details for a model element by UUID: all properties, exchanges, allocations. id (UUID)
search_elements Search elements by name pattern (substring or regex) across all layers. pattern, layer (optional), element_type (optional)
get_element_hierarchy Get the containment hierarchy tree for an element (ancestors and descendants). id (UUID), depth (default 3)
list_diagrams List all Sirius diagrams in the session with optional type and layer filters. layer (optional), diagram_type (optional)
list_requirements List requirements from the Requirements viewpoint with optional text filter. filter (optional text), module (optional)
get_traceability Get traceability links across ARCADIA layers for a given element. id (UUID), direction (upstream/downstream/both)
validate_model Run structural validation and return a list of issues grouped by severity. layer (optional, defaults to all layers)
get_allocation_matrix Get the function-to-component allocation matrix for a layer. layer (la/pa)
get_functional_chain Get functional chain details including all involved functions and exchanges. id (UUID, optional), name (optional search)
get_interfaces List interfaces with their exchange items, users, and involved components. layer (optional), component_id (optional)
get_scenarios List scenarios with instance roles and message sequences. layer (optional), scenario_type (optional)
explain_element Gather comprehensive context for AI explanation: full details, hierarchy, exchanges, allocations, diagrams. id (UUID)
get_state_machines List state machines with states, transitions, and trigger conditions. component_id (optional)
get_data_model List data types and classes in a layer (for data-flow models). layer
get_physical_paths List physical paths with their involved physical links and nodes. layer (pa default)
get_exchange_items List exchange items with their elements and types. layer (optional)
get_property_values Get all property values attached to a model element. id (UUID)
get_constraints List constraints with their expressions and constrained elements. layer (optional)
get_configuration_items List EPBS configuration items with their physical component realizations. none required
get_deployment_mapping Show PA behavior component-to-node deployment mapping. none required

Model Write Tools (21)

ToolDescriptionKey Parameters
create_element Create a function, component, or actor in the specified layer and parent. name, type (PhysicalComponent/PhysicalFunction/Actor/...), parent_id or parent_name, layer
create_exchange Create a functional or component exchange between two elements. source_id, target_id, name (optional), exchange_type
allocate_function Allocate a function to a component (creates FunctionRealization link). function_id, component_id
create_capability Create a capability with optional function involvements. name, layer, involved_function_ids (optional list)
update_element Update the name and/or description of an element by UUID. id (UUID), name (optional), description (optional)
delete_element Delete a model element. Requires explicit confirmation parameter. id (UUID), confirmed (must be true)
create_interface Create an interface with optional exchange items. name, layer, exchange_item_ids (optional)
create_functional_chain Create a functional chain with an ordered list of involved functions. name, layer, function_ids (ordered list)
create_physical_link Create a physical link between two node components or ports. source_id, target_id, name (optional)
batch_rename Batch rename elements whose names match a pattern (regex replace). layer, element_type, pattern, replacement
create_scenario Create an interaction scenario in a layer with instance roles. name, layer, scenario_type, instance_role_ids
create_state_machine Create a state machine on a component with optional initial state. component_id, name, initial_state (optional)
create_data_type Create a data type (class, enumeration, string, integer, etc.). name, type_kind, layer
move_element Move an element to a different parent container. id (UUID), new_parent_id
clone_element Deep-copy an element into the same parent (all children included). id (UUID), new_name (optional)
set_property_value Set a named string property value on an element. id (UUID), property_name, value
create_exchange_item Create an exchange item in a layer. name, layer, kind (FLOW/EVENT/SHARED_DATA/...)
batch_update_properties Set a property on multiple elements matching a filter at once. layer, element_type, filter, property_name, value
create_port Create a component or function port. parent_id, name, port_kind (FunctionInputPort/FunctionOutputPort/ComponentPort)
create_involvement Link a function to a capability (FunctionInvolvement). function_id, capability_id
create_generalization Create a generalization (inheritance) link between two components or actors. sub_id, super_id

Diagram Tools (9)

Tools for managing Sirius diagrams: update_diagram, refresh_diagram, create_diagram, export_diagram_image, show_element_in_diagram, list_diagram_elements, clone_diagram, auto_layout_diagram, delete_diagram.

Analysis Tools (13)

Tools for model health and quality analysis: check_traceability_coverage, run_capella_validation, detect_cycles, impact_analysis, find_unused_elements, model_statistics, allocation_completeness, get_refinement_status, interface_consistency, reachability_analysis, architecture_complexity, identify_singletons, generate_safety_report.

Export Tools (8)

Tools for generating reports and exports: export_to_csv, export_to_json, generate_icd_report, generate_model_report, export_traceability_matrix, export_allocation_matrix_csv, export_diagram_catalog, generate_diff_report.

Transition Tools (5)

Tools for ARCADIA layer transitions: transition_oa_to_sa, transition_sa_to_la, transition_la_to_pa, reconcile_layers, transition_pa_to_epbs.

AI Intelligence Tools (5)

Context-gathering tools that feed the LLM with structured model data for AI-driven recommendations: auto_allocate, suggest_interfaces, review_architecture, generate_test_scenarios, model_q_and_a.

Teamcenter Tools (7)

Note: Teamcenter tools are not fully implemented in the beta release. They return "not implemented" unless you have a customer-specific integration. See Known Limitations #9.

Available tools: search_teamcenter, tc_get_object, tc_get_bom, tc_list_requirements, tc_import_requirement, tc_import_part, tc_create_trace_link.

Simulation Tools (6)

Note: Simulation tools require MATLAB Engine API for Java. See Known Limitations #10.

Available tools: list_simulation_engines, extract_simulation_params, run_simulation, propagate_simulation_results, run_what_if, get_simulation_status.
