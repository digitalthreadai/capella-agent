Agent Modes

Each agent mode provides a specialized system prompt that tunes the AI's behavior for a specific engineering role. Switching modes changes:

  • The system prompt sent to the LLM at the start of the conversation.
  • The starter prompts shown on the empty-state card.
  • The default tool selection bias (which tools the agent is most likely to reach for first).

Switch modes via the toolbar dropdown or the /mode slash command.

Mode Overview

ModeStatusBest For
General Assistant Available Exploration, ad-hoc queries, learning the model
Sustainment Engineer Available Fault triage, ATA classification, impact analysis for maintenance engineers

General Assistant Mode

The default mode. The system prompt instructs the agent to act as a knowledgeable MBSE assistant familiar with ARCADIA, Capella, and systems engineering best practices. It makes no assumptions about the user's role and is suitable for any query.

Starter prompts:

  • "List all physical functions"
  • "Show me the functional chains in the PA layer"
  • "What are the interfaces between the Avionics and Cabin systems?"
  • "Run a model validation and summarize the issues"

Sustainment Engineer Mode

The system prompt is specialized for aerospace and defense sustainment workflows: fault triage, ATA chapter classification, impact radius analysis, and maintenance recommendation generation. Inspired by the Siemens Capella Days 2025 use case.

When you switch to this mode, the agent's reasoning is primed to think in terms of fault symptoms, component hierarchies, and field replacement units (FRUs).

Starter prompts:

  • "The Seat TV in row 14 is rebooting every time we try to play a VOD movie. Where should I look?"
  • "During an airline-imposed safety video broadcast, audio is silent on the cabin terminal but video plays fine. What functions are involved in audio routing for imposed broadcasts?"
  • "The Available VOD Movies List on the seat-back display is showing yesterday's titles. The Applications Server was rebooted. Which functions feed that list?"

See the full demo walkthrough in the Sustainment Demo cookbook recipe.

Requirements Analyst Mode (Week 13-16)

Planned for Weeks 13-16. This mode will specialize the agent for requirements engineering workflows:

  • Importing and exporting requirements via ReqIF and Excel.
  • Linking requirements to model elements.
  • Generating coverage dashboards showing which requirements have traceability links.
  • Detecting requirements without any downstream allocation (orphaned requirements).
  • Drafting requirements from natural language descriptions.

The four new tools shipping with this mode are: import_reqif, export_reqif, import_requirements_excel, and coverage_dashboard.

Architect Mode (Week 17-22)

Planned for Weeks 17-22. The most advanced mode — the agent acts as a generative architect that can:

  • Propose a complete architecture (components + functions + allocations) from a set of natural language requirements.
  • Generate candidate decompositions for a selected component.
  • Identify missing traces between layers and suggest how to fill them.
  • Present proposed changes as a diff preview before committing them to the model.
  • Explain its architectural reasoning in natural language.

Five new tools ship with Architect mode: propose_architecture, suggest_decomposition, find_missing_traces, generate_allocation_proposal, and review_architecture_diff.
