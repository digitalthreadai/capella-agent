Known Limitations

This page documents all known limitations in Capella Agent 1.0.0.beta1. Every item listed here was explicitly cut from the 6-week production plan after a 4-reviewer audit. Each entry explains why it shipped this way and when a fix is planned.

Detailed Notes

Limitation 1: Multi-Tool Compound Undo

When the agent performs multiple model changes in a single turn (e.g., "create three functions and link them"), each tool call commits its own RecordingCommand. A single Edit → Undo only reverses the most recent one. You need to undo N times to fully revert a multi-tool turn.

The fix — wrapping all tool calls in a single CompoundCommand per LLM turn — is approximately 5 days of work in the chat orchestration loop. It was deferred to v1.1 per the staff code reviewer recommendation.

Limitation 8: Code Signing

To unblock the ZIP on Windows:

  1. Right-click the downloaded ZIP file in Windows Explorer.
  2. Select Properties.
  3. At the bottom of the General tab, check Unblock.
  4. Click OK.
  5. Proceed with the installation.

Limitation 9: Teamcenter Integration

The com.capellaagent.teamcenter bundle builds and deploys, but the seven Teamcenter tools return "not implemented" because Teamcenter REST APIs are highly customer-specific (different server versions, different URL schemes, different security configurations). A generic implementation would fail in 90% of environments.

If your organization uses Teamcenter, contact CapellaAgent to scope a custom integration. Alternatively, Polarion REST is a more standardized interface that is a better first cross-tool bridge — it is on the product roadmap.
