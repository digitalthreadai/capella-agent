Recipe: Explore a Capella Model

This recipe teaches the fundamental navigation pattern for Capella Agent: listing elements, navigating by UUID, and querying relationships. Estimated time: 10 minutes.

Prerequisites: IFE sample model open, General Assistant mode active, a provider configured.

Step 1 — Open the IFE Model

Import the IFE project from demo/ into your workspace if you have not already. Open the .aird file to start the Sirius session. Confirm that the Project Explorer shows all 5 ARCADIA layers.

Step 2 — List All Physical Functions

In the chat input, type:

List all physical functions

The agent calls list_elements(layer=pa, element_type=functions) and returns a sortable HTML table with columns: Name, UUID, Type, and Description. The IFE model has roughly 30+ physical functions.

Try also: "List all system components in the SA layer" or "Show me all capabilities in the operational analysis".

Step 3 — Click a UUID to Navigate

In the result table, find a function that interests you — for example ProcessData or DisplayVideoOnSeatTV. Click the UUID in the table.

The Project Explorer automatically scrolls to and selects that element. The Eclipse Properties view at the bottom shows its full properties.

Step 4 — Get Full Element Details

Copy a UUID from the table (hover to reveal the copy icon). Then ask:

Get details for element 07256f31-32c7-42f5-b36b-bebd43492921

The agent calls get_element_details(id="07256f31-...") and returns a rich detail panel showing: full name, description, type, parent hierarchy, all outgoing and incoming exchanges, and any property values.

Step 5 — Show the Functional Chains

Show me the functional chains

The agent calls get_functional_chain for each chain it finds and returns a list. Each chain entry includes its name, UUID, and the sequence of involved functions.

Pick a chain from the list and ask for more detail: 

Show the full involvement sequence for the "Display VOD Movie on Seat TV" chain

Step 6 — Query Interfaces Between Systems

What are the interfaces between the Avionics and Cabin systems?

The agent calls get_interfaces filtered to exchanges between those two component groups. It returns a table of interfaces, their exchange items, and the direction (Avionics → Cabin or Cabin → Avionics).

Summary

You have learned the core navigation loop:

  1. List elements by layer and type.
  2. Click UUIDs to navigate to elements in the model.
  3. Ask for full details of any element.
  4. Query relationships: chains, interfaces, traceability.
