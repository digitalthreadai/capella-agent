Slash Commands

Slash commands are typed directly in the chat input field. They begin with a forward slash (/) and are processed immediately without sending anything to the LLM. Type /help at any time to see this list inside the chat view.

Command Reference

CommandAliasesDescriptionExample
/help /h, /? Display the slash command reference inside the chat view /help
/clear /c, /reset Clear the conversation history and return to the empty state. Does not affect model changes already made. /clear
/export /e, /save Export the current conversation to an HTML or Markdown file. Opens a file save dialog. /export
/mode /m Switch the active agent mode. Provide the mode name as an argument. Valid modes: general, sustainment, requirements, architect. /mode sustainment
/tools /t Show the full list of available tools with their categories and descriptions. The list is rendered as an HTML table in the chat area. /tools
/model — Show the currently active provider and model. Optionally switch model with an argument (must be a model supported by the active provider). /model
/model gpt-4o
/providers /p List all configured providers with their connection status (green/red) and the currently selected model for each. /providers
/undo /u Trigger Eclipse's undo action from within the chat. Equivalent to pressing Ctrl+Z in the Capella workbench. Useful after a model-modifying tool call. /undo
/version /v Display the Capella Agent version, build date, active provider, and the version of the connected Capella installation. /version

Notes

  • Slash commands are handled client-side in the plugin and do not consume any LLM tokens.
  • Arguments are separated from the command by a space: /mode sustainment.
  • Unknown commands are passed through to the LLM as regular text (the leading slash is preserved).
  • /clear does not undo any model changes. Use Edit → Undo or /undo to revert model changes.
Quick tip: After a long debugging session, use /clear to start fresh. The token counter drops to zero and you get the full context window back.
