Recipe: Requirements Roundtrip

Week 13-16 This recipe uses the Requirements Analyst agent mode and four tools that ship in Weeks 13-16: import_reqif, export_reqif, import_requirements_excel, and coverage_dashboard. The workflow described here will become available in that release.

The roundtrip: import requirements from ReqIF or Excel → link to model elements → view coverage → export back to ReqIF.

Step 1 — Switch to Requirements Analyst Mode

/mode requirements

The empty-state card updates to show requirements-focused starter prompts. The system prompt is now tuned for trace link creation and coverage analysis.

Step 2A — Import from ReqIF

import_reqif path="/path/to/your/requirements.reqif"

The agent parses the ReqIF file and imports the requirement modules and requirements into the Capella Requirements viewpoint. It returns a summary table:

  • Number of modules imported.
  • Number of requirements created.
  • Any requirements skipped (duplicate IDs or parse errors).

Step 2B — Import from Excel (Alternative)

import_requirements_excel path="reqs.xlsx" id_col="ID" text_col="Description" type_col="Type"

The agent reads the Excel file column-by-column using the provided column names. The id_col and text_col parameters are required; type_col, priority_col, and source_col are optional.

Step 3 — Link Requirements to Model Elements

Link a single requirement by its ID:

Link requirement REQ-IFE-047 to the "Display VOD Movie on Seat TV" functional chain

Or link in bulk using a natural language description:

Link all requirements containing "seat TV" to physical components in the Cabin System

The agent calls link_requirements_to_elements for each match and reports how many links were created.

Step 4 — View the Coverage Dashboard

Show me the requirements coverage dashboard

The agent calls coverage_dashboard and renders an HTML dashboard showing:

  • Overall coverage percentage (requirements with at least one trace link).
  • Coverage by module / type / priority.
  • A list of uncovered requirements (no links yet).
  • A list of model elements with no incoming requirements.

Step 5 — Export Back to ReqIF

Export all requirements to ReqIF at path "/output/updated_requirements.reqif"

The agent calls export_reqif and writes the updated ReqIF file with trace links included. The output is compatible with standard ReqIF tooling (IBM DOORS, PTC Integrity, Polarion).

Tip: After linking, run the coverage dashboard again to confirm coverage improved. Then ask: "Which requirements are still uncovered?" to get a focused list for the next linking session.
