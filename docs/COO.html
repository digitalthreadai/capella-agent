◆ Vision

“ Capella Agent bridges the gap between AI-assisted engineering and formal MBSE methodology. By embedding intelligent agents directly inside Eclipse Capella, engineers can query, modify, and validate architecture models using natural language — while maintaining full transactional safety, undo/redo support, and EMF model integrity.
⚙

In-Client AI

Runs inside Capella as a native Eclipse plugin, not as an external script or bridge. Full access to the live EMF model and Sirius sessions.

⧉

LLM Agnostic (9 Providers)

Claude, OpenAI, Groq, DeepSeek, Mistral, OpenRouter, Gemini, Ollama, or any custom endpoint — swap at runtime via OpenAiCompatibleProvider base class.

★

PLM Connected

Direct Teamcenter REST integration for importing requirements, parts, BOMs, and creating bidirectional trace links.

▷

Simulation Ready

Extract parameters from model, run MATLAB/Simulink simulations, propagate results back, and perform what-if analysis.

▷

Claude Code MCP

MCP Server bridge lets you use Claude Code terminal as an alternative AI brain — same 29 tools, live model access, zero extra config.

⧉ Key Interfaces

Core abstractions that enable extensibility and provider swapping

ILlmProvider

Abstracts LLM vendor APIs. Implementations for Claude, OpenAI, Gemini, and Ollama. Supports streaming responses and tool-use protocol.

IToolDescriptor / IToolExecutor

Separates tool metadata (name, schema, description) from execution logic. Enables dynamic tool discovery and registration.

AbstractCapellaTool

Base class for all Capella model tools. Provides transaction wrapping, model access helpers, and error handling scaffolding.

ISimulationEngine

Contract for simulation backends. Implementations connect to MATLAB Engine API, Simulink, or custom solvers.

IAgentMessageBus

Pub/sub message bus enabling inter-agent communication. Agents can delegate tasks and share context without tight coupling.

ToolRegistry

Central registry for all tools across agents. Supports filtering by category, access level, and agent scope. OSGi service-based.

☐ Data Layer

EMF-native persistence with transactional safety

☷

No Traditional Database

Uses EMF (Eclipse Modeling Framework) in-memory model backed by XMI/AIRD files. The Capella model IS the database — all reads and writes go through EMF resource sets.

✓

TransactionalEditingDomain

All write operations are wrapped in EMF transactions via TransactionalEditingDomain. Ensures atomicity — partial changes are rolled back on failure.

↺

RecordingCommand

Every tool-initiated change uses RecordingCommand for full undo/redo support. Users can reverse any AI-made modification with Ctrl+Z.

◆

Sirius Session

Diagram access goes through the Sirius Session, which manages the mapping between semantic model elements and their graphical representations.

⚿ Security

Defense-in-depth approach for model and credential safety

⚿

Access Control

READ_ONLY mode by default. Engineers must explicitly switch to READ_WRITE before any model modifications. Prevents accidental changes from AI operations.

🔒

Credential Storage

API keys and Teamcenter credentials stored in Eclipse Equinox Secure Preferences with OS-level encryption. Never persisted in plain text or project files.

📋

Audit Logging

All tool executions are logged with timestamp, action type, element IDs, and execution result. Full audit trail for compliance and debugging.

☰ Roadmap

13-week implementation plan across 5 phases

Phase 1: Foundation
Weeks 1 – 3
Core plugin structure, LLM provider interface, tool registry, chat UI, EMF model access layer.
Phase 2: Model Chat
Weeks 3 – 6
16 model tools (read, write, diagram), transaction safety, undo/redo, validation integration.
Phase 3: Teamcenter
Weeks 6 – 8
REST client, search/import tools, trace links, credential management, PLM sync workflows.
Phase 4: Simulation
Weeks 8 – 11
MATLAB Engine bridge, parameter extraction, result propagation, what-if analysis, status monitoring.
Phase 5: Integration
Weeks 11 – 13
Multi-agent orchestration, message bus, cross-agent workflows, testing, documentation, release.
