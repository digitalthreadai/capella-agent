Setup & Configuration Guide

Step-by-step guide to build, install, and configure Capella Agent for AI-powered MBSE workflows.

1

Prerequisites

~5 min

Ensure the following tools are installed and available on your system PATH before proceeding.

requirements 
Required:
  Eclipse Capella 7.0+     https://mbse-capella.org/download.html
  Java JDK 17+             https://adoptium.net/
  Maven 3.9+               https://maven.apache.org/

Optional:
  MATLAB R2022b+           For simulation agent
  Teamcenter AW Server     For PLM integration
  Ollama                   For local LLM (no API key needed)
2

Build

~10 min

Clone the repository and build all Eclipse plugins using the Tycho/Maven build system.

bash 
# Clone the repository
git clone <repo-url>
cd capella-agent

# Build all plugins with Tycho
mvn clean verify -f pom.xml

# Output: com.capellaagent.site/target/repository/
ⓘ First build may take several minutes to download Capella target platform dependencies. Subsequent builds will be significantly faster.
3

Install in Capella

~5 min

Install the built plugins into your Eclipse Capella installation using the local update site.

  1. Open Eclipse Capella
  2. Navigate to Help > Install New Software...
  3. Click "Add..." > "Local..." > Browse to com.capellaagent.site/target/repository
  4. Check the "Capella Agent" feature
  5. Click Next > Accept license agreement > Finish
  6. Restart Capella when prompted
4

Configure LLM Provider

~5 min

Select and configure your preferred Large Language Model provider. Claude is recommended for best results.

◆ Claude by Anthropic

Best performance for MBSE tasks. Requires an API key from Anthropic.

  1. Get your API key from console.anthropic.com
  2. In Capella, go to Preferences > Capella Agent > LLM Provider
  3. Set Provider to anthropic
  4. Set Model to claude-sonnet-4-20250514
  5. Paste your API key (stored securely in Eclipse Secure Storage)
◆ OpenAI

GPT-4o support with optional Azure OpenAI endpoint override.

  1. Get your API key from platform.openai.com
  2. In Capella, go to Preferences > Capella Agent > LLM Provider
  3. Set Provider to openai
  4. Set Model to gpt-4o
  5. Paste your API key
ⓘ For Azure OpenAI deployments, set the endpoint override in the advanced settings to your Azure resource URL.
◆ Groq Cloud

Ultra-fast inference on custom LPU hardware. Free tier available. Also supports DeepSeek, Mistral, OpenRouter via the same OpenAI-compatible pattern.

  1. Get your API key from console.groq.com
  2. In Capella, go to Preferences > Capella Agent > LLM Provider
  3. Set Provider to groq
  4. Set Model to llama-3.3-70b-versatile
  5. Paste your API key
ⓘ Other OpenAI-compatible providers (DeepSeek, Mistral, OpenRouter) follow the same pattern — just select the provider ID and enter the API key. Use custom for any unlisted endpoint.
◆ Ollama (Local, Free)

Run models locally with no API key or cloud dependency. Great for air-gapped environments.

  1. Install Ollama from ollama.com
  2. Pull a model:
    bash 
    ollama pull llama3.1
  3. Set Provider to ollama
  4. No API key needed
  5. Endpoint: http://localhost:11434
5

Configure Teamcenter (Optional)

~10 min

Connect to Siemens Teamcenter for PLM data synchronization and traceability.

  1. Navigate to Preferences > Capella Agent > Teamcenter
  2. Set Gateway URL: https://your-tc-server/tc
  3. Select authentication method: Basic or SSO
  4. Enter username and password (stored in Eclipse Secure Storage)
  5. Click "Test Connection" to verify connectivity
⚠ Actual REST endpoint paths depend on your Teamcenter/Active Workspace version. Consult your Teamcenter administrator for the correct gateway URL.
6

Configure Simulation (Optional)

~5 min

Enable MATLAB-based simulation for model validation and co-simulation workflows.

  1. Navigate to Preferences > Capella Agent > Simulation
  2. Set MATLAB executable path, e.g.: C:\Program Files\MATLAB\R2024a\bin\matlab.exe
  3. Set working directory, e.g.: C:\temp\capella-sim
  4. Set timeout: 300 seconds (default)
7

Security & Access Control

~2 min

Capella Agent includes built-in safety mechanisms to protect your model data.

◯ READ_ONLY Mode

Default mode. Safely explore and query models without risk of modification. Recommended for initial setup.

✎ READ_WRITE Mode

Enable when you need the agent to create or modify model elements. All changes are tracked in the audit log.

📋 Audit Log

All operations logged to:
workspace/.metadata/.plugins/com.capellaagent.core/audit.log

🔒 Secure Storage

All API keys encrypted via Eclipse Equinox Secure Storage. Credentials never stored in plain text.

8

Claude Code MCP Integration (Optional)

~5 min

Use Claude Code terminal as an alternative AI brain. The MCP plugin starts automatically when Capella loads, exposing all 29 tools via localhost HTTP. A bridge subprocess translates MCP stdio to HTTP.

  1. Install Claude Code and ensure claude is on your PATH
  2. Ensure Capella is running with a model open (MCP endpoint auto-starts on port 9847)
  3. Open a terminal in the capella-agent directory (where .mcp.json lives)
  4. Run claude — it discovers the MCP tools automatically
  5. Ask: "List all system functions in the SA layer"
.mcp.json 
{
  "mcpServers": {
    "capella-live": {
      "command": "java",
      "args": ["-cp", "com.capellaagent.mcp/bin",
               "com.capellaagent.mcp.bridge.McpServerBridge", "9847"]
    }
  }
}
ⓘ Set CAPELLA_MCP_PORT before launching Capella to use a different port. Update the port in .mcp.json to match. The 9 LLM providers (Groq, DeepSeek, Mistral, etc.) are for the in-Eclipse chat only — Claude Code uses its own LLM.
9

Verification

~3 min

Verify that Capella Agent is installed and configured correctly with a quick end-to-end test.

verification steps 
1. Open a Capella project with .aird model
2. Window > Show View > Capella Agent > AI Model Chat
3. Type: "List all system functions"
4. Verify response shows model elements
5. Check audit log for entries
ⓘ If the AI Model Chat view is not visible, ensure the Capella Agent plugins were installed correctly and Capella was restarted.

Troubleshooting

"Build fails"
Verify that Java 17+ is installed and JAVA_HOME is set correctly. Run java -version to confirm. Ensure Maven 3.9+ is on your PATH and that the Capella target platform dependencies can be resolved. If behind a proxy, configure Maven proxy settings in ~/.m2/settings.xml.
"No response from LLM"
Check that your API key is valid and correctly entered. Verify network connectivity to the provider endpoint. For Ollama, ensure the service is running (ollama serve). Check the Eclipse Error Log view (Window > Show View > Error Log) for detailed error messages.
"Teamcenter connection refused"
Verify the gateway URL is correct and the Teamcenter Active Workspace server is running. Check credentials and authentication method (Basic vs. SSO). Ensure network access to the server is not blocked by firewalls. Test the URL directly in a browser first.
"MATLAB not found"
Verify the MATLAB executable path in preferences points to the correct matlab.exe (Windows) or matlab (Linux/macOS) binary. Ensure the MATLAB installation is R2022b or later. Check that the working directory exists and is writable.
